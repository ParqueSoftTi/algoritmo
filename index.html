Objetivo

  • Entender los conceptos básicos de la complejidad algorítmica.
  • Aprender a identificar la complejidad de los algoritmos mediante ejemplos en PSeInt.

Teoría

  • Complejidad Temporal: Mide el tiempo que tarda un algoritmo en ejecutarse.
  • Complejidad Espacial: Mide la cantidad de memoria que necesita un algoritmo.

Notación Big-O

  • O(1): Complejidad constante.
  • O(n): Complejidad lineal.
  • O(n^2): Complejidad cuadrática.
  • O(log n): Complejidad logarítmica.

Ejemplo 1: Complejidad O(1) - Constante

Algoritmo ComplejidadConstante
    Definir x Como Entero
    x = 10
    Escribir "El valor de x es: ", x
FinAlgoritmo

Explicación: El algoritmo realiza una operación que toma el mismo tiempo independientemente del tamaño de la entrada.

Ejemplo 2: Complejidad O(n) - Lineal

Algoritmo ComplejidadLineal
    Definir n, i Como Entero
    Escribir "Ingrese un número: "
    Leer n
    Para i = 1 Hasta n Hacer
        Escribir "Número: ", i
    Fin Para
FinAlgoritmo

Explicación: El algoritmo realiza una operación n veces, donde n es el tamaño de la entrada.

Ejemplo 3: Complejidad O(n^2) - Cuadrática

Algoritmo ComplejidadCuadratica
    Definir n, i, j Como Entero
    Escribir "Ingrese un número: "
    Leer n
    Para i = 1 Hasta n Hacer
        Para j = 1 Hasta n Hacer
            Escribir "Par (", i, ", ", j, ")"
        Fin Para
    Fin Para
FinAlgoritmo

Explicación: El algoritmo realiza una operación n * n veces, resultando en una complejidad cuadrática.

Ejemplo 4: Complejidad O(log n) - Logarítmica

Algoritmo ComplejidadLogaritmica
    Definir n, contador Como Entero
    Escribir "Ingrese un número: "
    Leer n
    contador = 0
    Mientras n > 1 Hacer
        n = n / 2
        contador = contador + 1
    Fin Mientras
    Escribir "Número de divisiones: ", contador
FinAlgoritmo

Explicación: El algoritmo divide n por 2 en cada iteración, resultando en una complejidad logarítmica.
